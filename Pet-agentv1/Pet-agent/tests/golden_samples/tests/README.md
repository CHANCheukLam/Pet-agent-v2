# Test Cases – Phase 2 (Single‑Agent Pre‑Trip)

This folder contains deterministic golden test outputs for the
single‑agent **pre‑trip planning pipeline**.

All JSON files are generated by running the evaluation harness and
are intended for manual inspection and evaluation by reviewers.

---

## Test cases

- **case_001_cn_happy**  
  Complete Chinese pre‑trip route planning (Beijing → Shanghai, all required fields provided).  
  Expected: `success`.

- **case_002_en_happy**  
  Complete English pre‑trip route planning with the same intent and constraints.  
  Expected: `success`.

- **case_003_hard_block**  
  Overweight pet (large dog) violates transport constraints.  
  Expected: `blocked_hard_constraint` with one clear refusal reason.

- **case_004_missing_fields**  
  Input: “我想带狗去上海玩” with missing dates, duration, and transport information.  
  Expected: `blocked_incomplete_context` with one follow‑up question.

- **case_005_fallback_static_only**  
  Dynamic routing and weather data unavailable; the system falls back to static rules only.  
  Expected: `success` with structured output and dynamic sources marked as unavailable.

- **case_006_weather_mock**  
  Hot‑weather scenario using mock weather data; outdoor activity is constrained.  
  Expected: `success` with weather‑related constraints reflected in planner input.
